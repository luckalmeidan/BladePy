Detailed Description

This is the key Class that wraps all the other packages and modules of BladePy.

This class is responsible for adding functions to the inherited output_viewerUI.Ui_MainWindow function-less layout created in Qt Designer.

This class has methods for managing the display of the outputs generated by BladePro. It has the methods for loading any case that was previously generated by BladePro. It can also retrieve the outputs of BladePro as soon as they were created if desired in the bladepro_modules.inputfile_writer.InputWriterWindow

Classes of other BladePy modules of objects created by BladePyCore (composition association)

For output display:

  • bladepro_modules.inputfile_writer.InputWriterWindow - For writing inputfiles and BladePro communication
  • tecplot_modules.tecplot_display.TecPlotWindow - For displaying tecplot graphics
  • occ_modules.qt_display.customQtViewer3d - For creating a customized viewer for OpenCascade
  • occ_modules.shape_properties.customQtViewer3d - For creating a customized viewer for OpenCascade

For data management:

  • data_structure.case_model.CaseModel - For creating a model for TreeView list
  • data_structure.case_node.CaseNode - For creating a information center for cases

For user settings:

  • settings.preferences.PreferencesBladePy - For managing User Preferences

def Core.BladePyCore.addCase (   self,
  object 
)

Method that adds a Case in Output Viewer.

This will be triggered by the Output Viewer when opening an existing case or when Run BladePro button is clicked in in the Input Writer GUI. This method will read files related to the working case. Currently, the addCase method supports three outputs from BladePro.

  • IGS 3D Curves;
  • IGS Surfaces;
  • Tecplots 2D.
